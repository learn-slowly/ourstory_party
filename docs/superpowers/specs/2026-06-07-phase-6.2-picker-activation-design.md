# ourstory Phase 6.2 — emd · station picker 활성화 설계

작성일: 2026-06-07
대상 repo: `learn-slowly/ourstory_party`
선행: `318fefb` (sido/sigungu picker client-fetch fix), `4f97d8b` (AdvancedTable 도입)

## 동기

홈 picker 의 emd(읍·면·동) · station(투표소) 단계는 force-static + outputFileTracingExcludes 한계로 비활성 상태. 그러나 실제 데이터는 충분: region 파일 4,216개 (emd 단위 포함) · station 파일 29,884개 · `hasStationLevel=true` 선거 21개 (2002 대선·2004 총선 등). 사용자 needs(예: "창원 성산구 상남동 상남제3투의 2004 권영길 ~ 2025 대선 추이") 가 데이터 측면에선 이미 가능. picker UI 만 활성화하면 됨.

이미 sido/sigungu 는 client-fetch 로 해결(2afb53b + 318fefb). 같은 패턴을 emd/station 까지 확장.

## 범위

- `build:static` 산출물의 `index.json` 에 `stationListByEmd?: Record<emdCode, string[]>` 추가
- HomeView 가 emd 옵션을 `index.regions.emdByRegion[sigunguCode]` 에서 useMemo 로 도출
- HomeView 가 station 옵션을 `stationListByEmd[emdCode]` 에서 useMemo 로 도출
- station 선택 시(`state.region` 이 `station:...` 형식) `/data/static/station/{stationKey}.json` client fetch
- HeaderControls 의 emd/station select 활성/비활성 조건 갱신
- 정적 호스팅 (force-static + outputFileTracingExcludes) 그대로 유지

## 비목표

- 공유 URL 로 picker 상태 복원 — mount 시 client URL 파싱. 별도 후속 (사용자 직접 진입 시 picker 가 "전국" 으로 시작하는 한계 잔존)
- region/[code] 페이지의 ElectionPicker 동일 문제 해결 — 별도 후속
- 신규 region 단계 추가 — 시·도/시·군·구/읍·면·동/투표소 그대로

## 의사결정 요약

| 항목 | 결정 | 이유 |
|---|---|---|
| station 옵션 데이터 소스 | `index.json` 에 `stationListByEmd` 추가 | 별도 endpoint 보다 단일 index 가 client 흐름 단순. `emdByRegion` 와 같은 위치 |
| station 키 형식 | 기존 그대로 (`{sigunguName}-{emdName}-{stationName}.json`) | build·HomeView 모두 이미 그 형식 사용 |
| picker 활성/비활성 조건 | emd: `emdByRegion[sigungu]` 비어있지 않으면 활성. station: `stationListByEmd[emd]` 비어있지 않으면 활성 | hasStationLevel 은 election 메타. picker 활성화는 region 메타 — 분리 |
| station 데이터 없는 election 처리 | timeseries 의 빈 entry → cell `null` → 시계열에 "—" | 기존 미출마 처리 패턴과 동일 |
| build 산출물 크기 | `stationListByEmd` 는 ~30k stations × short string. 추정 ~2MB. 압축하면 ~500KB | index.json 이 한 번 로드되는 SSG 패턴이라 허용 |

```
│  ├─ public/data/static/station/*.json (29,884개) — 그대로        │
│  └─ public/data/static/index.json                                │
│      ├─ elections, parties, regions.sido, regions.sigunguByRegion│
│      ├─ regions.emdByRegion  (기존)                              │
│      └─ regions.stationListByEmd  ◀── 신규                       │
└─────────────────────────────────────────────────────────────────┘
                          │
                          ▼ (force-static prerender, public CDN)
┌─────────────────────────────────────────────────────────────────┐
│ page.tsx (RSC, force-static)                                     │
│  └─ HomeView props: { index, ... }                               │
└─────────────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────────┐
│ HomeView (client)                                                │
│  ├─ state.region cascading: all → sido → sigungu → emd → station│
│  ├─ emdOptions = useMemo(emdByRegion[sigunguCode])               │
│  ├─ stationOptions = useMemo(stationListByEmd[emdCode])          │

│  │   · "all" → initial sources                                   │
│  │   · 10자리 code(sido/sigungu/emd) → region/{code}.json        │
│  │   · "station:S:E:N" → station/{sigunguName}-{emdName}-{N}.json│
│  └─ AdvancedTable + 차트 같음                                    │
└─────────────────────────────────────────────────────────────────┘
```

## 데이터 흐름

1. **빌드 시**: `build:static` 이 station 파일명 29,884개를 sigunguName/emdName 으로 group → `stationListByEmd[emdCode]` 채움. emdCode 매핑은 region/{emdCode}.json 의 parent + self 이름.
2. **첫 렌더**: page.tsx (force-static) 가 prerender 결과 — 전국 기준 sources 만.
3. **사용자가 시·도 선택**: 기존 client-fetch — region/{sidoCode}.json
4. **시·군·구 선택**: region/{sigunguCode}.json
5. **읍·면·동 picker 활성**: `stationListByEmd` 대신 emd 는 `emdByRegion[sigunguCode]` 에서 도출 — region/{emdCode}.json fetch
6. **투표소 picker 활성**: `stationListByEmd[emdCode]` 에서 도출 — station/{stationKey}.json fetch
7. **시계열**: hasStationLevel=true 선거만 의미 있는 데이터, 나머지는 빈 entry (cell null → "—" 표시)

## 에러·엣지케이스

| 케이스 | 처리 |
|---|---|
| stationListByEmd 의 emdCode 매핑 누락 | 그 emd 의 station picker 비활성 (빈 배열) |
| station 파일 fetch 실패 (404) | setClientSources([]) → chart 빈 상태 메시지 |
| hasStationLevel=false election + station 단위 | 그 election entry 의 cells 가 null → 시계열에 점 없음 (gap) |
| sigunguName/emdName 의 특수문자(슬래시·콜론·따옴표) | 이미 `static-data.ts` 의 `safe = stationKey.replace(/[\/\\]/g, "_")` 처리 — 빌드도 동일 normalize |
| 다중 sigungu 동명 emd (예: 여러 시·군·구의 "중앙동") | emdCode 가 유일하므로 problem 없음 — emdByRegion 키는 sigungu code |
| stationOptions 가 너무 많은 emd (수십~수백 station) | scrolling select. 추후 검색 가능한 typeahead 는 별도 후속 |
