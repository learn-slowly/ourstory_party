# ourstory Phase 7.1 — 2022 8회 지선 읍·면·동 데이터 확장 설계

작성일: 2026-06-07
대상 repo: `learn-slowly/ourstory_party`
선행: 정적 마이그레이션(`0b9e5714`), `build:parse` xlsx 파서 (`scripts/build/parse-nec-xlsx.ts`)
입력: `~/Downloads/제8회_전국동시지방선거_읍면동별_개표결과-게시판게시.zip`

## 동기

지방선거는 NEC archive 가 시·군·구 단위까지만 공개해 ourstory 도 그 수준에 머묾. 사용자 needs(예: "창원 성산구 상남동의 정당 변화") 충족 위해선 emd 단위 데이터가 필요. NEC 가 별도 게시판으로 공개한 **"읍·면·동별 개표결과"** zip 을 파싱해 region.json 의 timeseries 에 emd 단위 entry 를 추가.

본 phase 는 2022 8회 지선만 (사용자 사례에 가장 핵심 — 가장 최근). 같은 패턴으로 7.2(2018) · 7.3(2014) 후속.

## 범위

- `scripts/build/parsers/parse-jiseon-2022.ts` 신규 — zip 안 xlsx 9개(선거유형별) 파싱
- 정당명 통합 (`data/meta/parties.json` 또는 동일 alias 메커니즘)
- region 코드 매핑 (시·도 / 시·군·구 / 읍·면·동 이름 → 10자리 코드)
- 결과를 `data/parsed/` 의 중간 산출물 (json) 로 저장
- `build:parse` 가 본 파서 호출 — 기존 xlsx parser 와 같은 통합점
- `build:static` 이 중간 산출물을 region.json 의 timeseries 에 합산
- `index.json` 의 `elections` 에 2022 지선 7개 election 들의 emd 단위 보강 표시 (이미 있을 수도 — 검증)

## 비목표

- 2018·2014 지선 — 후속 phase 7.2 / 7.3
- 옛 대선·총선 zip — 후속 phase 7.4

## 의사결정 요약

| 항목 | 결정 | 이유 |
|---|---|---|
| 파서 위치 | `scripts/build/parsers/parse-jiseon-2022.ts` (단독 파일) | 기존 `parse-nec-xlsx.ts` 는 다른 형식. 형식별 어댑터 분리 |
| 인코딩 | CP949 → UTF-8 (filename / cell value) | 한글 깨짐 방지. `iconv-lite` (이미 dep) |
| zip 처리 | `adm-zip` 또는 `unzipper` 새 dep | 7M zip 안 9 xlsx — 메모리 안에서 풀고 처리 |
| xlsx 파싱 | 기존 `xlsx` (SheetJS, 이미 dep) | sheet → 2차원 행으로 |
| 정당명 통합 | parties.json alias 기존 메커니즘 | 새 정당 발견 시 추가 |
| region 이름 → 코드 lookup | `data/meta/region-name-to-code.json` (build 시 index 에서 도출) | 이름이 시·도/시·군/읍·면·동 중복 (중앙동 등) — 부모 경로 함께 |
| 시·도/시·군/읍·면·동 이름 정규화 | 공백 트림 + "전체" / "소계" / "합계" 행은 skip | NEC xlsx 의 footer/header 노이즈 처리 |
| 미정상 행 처리 | log + skip | 매핑 실패 행은 카운트해 검증 |

- 헤더 (1~3행): 시·도 / 시·군·구 / 읍·면·동 / 후보자명 + 정당 / 득표수
- 본문: emd 행 단위로 정당별 득표수
- 정당 컬럼 동적 (선거유형별 다름)

## 데이터 흐름

```
~/Downloads/...zip
       │
       ▼
parse-jiseon-2022.ts:
  ├─ adm-zip 으로 9 xlsx 추출 → in-memory ArrayBuffer
  ├─ iconv-lite 로 한글 filename 디코드
  ├─ xlsx.read 로 각 xlsx 파싱
  ├─ 행별 (시·도, 시·군, 읍·면·동, 정당, 득표수) 추출
  ├─ region-name-to-code 로 10자리 emd code 매핑
  ├─ parties.json alias 로 정당 정규화
  └─ 결과: data/parsed/2022-local/{electionId}.json
       (rows: [{ regionCode, partyId, votes, totalVotes, share }])
       │
       ▼
build-static.ts:
  ├─ 기존 흐름 + 신규 step:
  ├─ data/parsed/2022-local/*.json 읽어
  ├─ 각 row 의 regionCode → region/{code}.json 찾아 timeseries 에 entry 추가:
  │     timeseries[partyId].push({ electionId, votes, totalVotes, share })
  ├─ index.json 의 elections 에 이미 있는 2022 지선 election 들 그대로 (메타 변화 X)
  └─ public/data/static/ 산출
```

## 에러·엣지케이스

| 케이스 | 처리 |
|---|---|
| zip 안 xlsx 파일명이 예상과 다름 | 파서 첫 부분에 expected filenames vs actual 명시. 차이 시 명확한 에러 |
| region 이름이 시·도/시·군/읍·면·동에서 중복 (중앙동 등) | 부모 (시·군) 함께 lookup. 그래도 충돌 시 unmapped 카운트 + log |
| 정당명이 alias 에 없음 | "other" 로 grouping + 카운트 + log. 일정 % 이상이면 build 실패 |
| 시·도/시·군 단위 합계 행 (예: "전국 합계", "경상남도 합계") | header 행과 동일 패턴 — skip 조건에 포함 |
| 거소·관외사전·재외 의 외부 row 처리 | emd 가 아닌 "선거 외 분류" 라 region.json 안에 안 들어감. 별도 폴더 (`data/parsed/2022-local/external/`) 로 저장 (향후 활용) |
| 빌드 시간 증가 | 9 xlsx × 수만 행 → 수십초 추정. 첫 실행만 — 이후 cache 가능 (mtime 비교) |

## 테스트 전략

**Unit (Vitest)**
- `parse-jiseon-2022.test.ts`:
  - 정상 행 → 올바른 emd code 매핑
  - "합계" / "전국" row skip
  - alias 통합 (예: "더불어민주당" → democratic)
  - 부모 경로로 중앙동 disambiguation
  - 정당 컬럼이 동적인 xlsx → 정상 추출

**Integration (manual)**
- 본 파서를 실제 zip 으로 1회 실행
- `data/parsed/2022-local/` 에 9개 json 생성 확인
- 행 수 = NEC 공식 발표 합과 비교 (검증 spreadsheet 만들기)

**Smoke (build-static 통합 후)**
- 빌드 후 region/4811800000.json (성산구 같은 sigungu) 의 timeseries 에 2022 지선 entry 있는지
- 그 region 의 자식 emd code 의 region.json timeseries 에 2022 지선 entry 있는지
- 홈 picker 에서 경남 → 창원시 → 상남동 → 2022 8회 지선 정의당 값 표시

### 통과 기준

- 단위 테스트 5+ PASS
- 빌드 PASS, 매핑 실패 행 < 0.1%
- 사용자가 "창원 성산 상남동 정의당 2022 지선 광역단체장" 시계열 점 확인 가능
